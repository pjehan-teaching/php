Conditions

Introduction

Les conditions permettent d'exécuter une ou plusieurs instructions en fonction du résultat d’une comparaison.

Exemple : Si l’utilisateur a posté plus de 50 messages, alors c’est un utilisateur VIP. Sinon, il s’agit d’un utilisateur débutant.

Introduction

On pourrait représenter l’algorithme précédent avec le code PHP suivant :

<?php
$nbMsg = 105;

if($nbMsg > 50) {
  echo 'Utilisateur VIP';
} else {
  echo 'Utilisateur débutant';
}
?>

Symboles de comparaison

Les symboles de comparaison :

Symbole Description
== Est égale à
> Est supérieur à
>= Est supérieur ou égale à
< Est inférieur à
<= Est inférieur ou égale à
!= Est différent de
Les symboles === et !== permettent également de comparer le type de donnée Le symbole "=" change la valeur de la variable

Symboles de comparaison

Dans l’algorithme suivant, quelles conditions sont respectées ?

<?php
$nbMsg = 2;
if($nbMsg == 2) { }
if($nbMsg <= 2) { }
if($nbMsg > 2) { }
if($nbMsg !== '2') { }
?>

Symboles de comparaison

Avec les variables de type booléen, on peut utiliser une syntaxe simplifiée :

<?php
$isAdmin = true;

if ($isAdmin == true) { }
if ($isAdmin == false) { }

// Syntaxe simplifiée
if ($isAdmin) { }
if (!$isAdmin) { } // Ici, le point d'exclamation permet d'inverser la condition
?>

Symboles de comparaison

On peut également combiner plusieurs conditions :

<?php
$nbMsg = 105;
$registrationDate = new DateTime("2015-07-08 11:14:15");
$isAdmin = false;

// Nombre de jours depuis l'inscription de l'utilisateur
$registrationDays = $registrationDate->diff(new DateTime())->days;

if($isAdmin) {
  echo 'Administrateur';
} else if ($nbMsg > 50 && $registrationDays < 30) {
  echo 'Flood';
} else if ($nbMsg > 50 || $registrationDays >= 365) {
  echo 'Utilisateur VIP';
} else {
  echo 'Utilisateur débutant';
}
?>

Symboles de comparaison

Symbole Symbole alternatif Description
&& AND Et
|| OR Ou
Il est préférable d’utiliser la 1ère syntaxe qui est commune à la plupart des langages

Le switch

Lorsqu'une variable est comparée à de nombreuses valeurs, il est possible d'utiliser un switch :

<?php
if ($nbJoueurs == 1) {
  echo "Solitaire";
} else if ($nbJoueurs == 2) {
  echo "Tennis";
} else if ($nbJoueurs == 4) {
  echo "Volleyball";
} else if ($nbJoueurs == 10) {
  echo "Basket-ball";
} else if ($nbJoueurs == 14) {
  echo "Handball";
} else {
  echo "Football";
}
?>
<?php
switch ($nbJoueurs) {
  case 1:
    echo "Solitaire";
    break;
  case 2:
    echo "Tennis";
    break;
  case 4:
    echo "Volleyball";
    break;
  case 10;
    echo "Basket-ball";
    break;
  case 14:
    echo "Handball";
    break;
  default:
    echo "Football";
}
?>

Le switch

Attention cependant à ne pas oublier l'instruction break; !

<?php
$nbJoueurs = 2;

switch ($nbJoueurs) {
  case 1:
    echo "Solitaire";
    break;
  case 2:
    echo "Tennis";
  case 4:
    echo "Volleyball";
    break;
  case 10;
    echo "Basket-ball";
    break;
  default:
    echo "Football";
}
?>

Le code ci-contre affichera le texte "TennisVolleyball"

Opérateur ternaire

Il existe également une autre façon d’écrire des conditions appelée condition ternaire :

<?php $message = ($score > 10) ? "Vous avez gagné" : "Vous avez perdu"; ?>
Il est préférable d’éviter d’utiliser cette syntaxe pour plus de clarté dans le code source

Exercice badge

Règles de gestion :

  1. Si l’utilisateur est admin, afficher le badge "Administrateur".
  2. Si l’utilisateur est un homme, afficher le badge “Homme”. Sinon, afficher celui indiquant qu’il s’agit d’une femme.
  3. Utilisez des instructions “if” pour afficher :
    • Une étoile vide si l’utilisateur a posté moins de 50 messages.
    • Une étoile à moitié remplie si l’utilisateur a posté entre 50 et 100 messages.
    • Une étoile pleine si l’utilisateur a posté plus de 100 messages.
  4. Si l’utilisateur est inscrit depuis moins de 30 jours et qu’il a posté plus de 50 messages, afficher le badge "Flood".
  5. Si l’utilisateur est inscrit depuis plus de 365 jours ou qu’il a posté plus de 100 messages (mais qu’il n’a pas la badge "Flood"), afficher le badge "VIP".
  6. Si la date du jour correspond à la date de naissance de l’utilisateur, afficher le badge "Anniversaire".
